The 
exec
Family of Calls

  • A family of 
    exec
    functions are built on and internally invoke a single syscall (
    execve()
    ). Here is one family member: 
    int execl (const char *path, const char *arg, ...);
  • It loads the code of the program whose name is given by 
    path
    into main memory. The argument 
    arg
    should contain the name of the new child program.
  • The ellipsis (
    ...
    ) represents variadic command-line arguments to the child program, which must end with 
    NULL
    . Example: 
    int ret = execl ("/bin/vim", "vim", "text.txt", NULL); // Same as typing: vim text.txt.
if (ret == -1) // Error checks.
{
    perror ("execl");
    exit (EXIT_FAILURE);
}
