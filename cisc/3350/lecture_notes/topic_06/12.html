Memory-Mapped I/O: The System Calls

Name: 
mmap()
 System call or function: System Call Links: Online Manual | Course Packet
What it does: maps (copies) the contents of the file 
fd
to main memory.
What libraries you must include: 
#include <sys/mman.h> // Defines mmap().
Syntax: 
void * mmap (void *addr, size_t len, int prot, int flags, int fd, off_t offset);
Description of arguments: 
addr
: a hint to OS where to map file. Pass 
0
 for the OS to decide itself.

len
: number of bytes you want to copy from the file represented by 
fd
.

prot
: memory protection mode: one or more of 
PROT_READ
, 
PROT_WRITE
, and 
PROT_EXEC
.

flags
: type of mapping: one of 
MAP_FIXED
, 
MAP_PRIVATE
, 
MAP_SHARED
, etc.

fd
: file descriptor of an open file for writing.

offset
: offset of bytes (= how deep inside the file you want to start copying.)
What type it returns: 
void*
 On success: an address (pointer) in memory to the beginning of the file's content.
On failure: 
MAP_FAILED
(= 
((void*) -1)
)
If failure, does it set 
errno
? Yes
Quick example: 
if ((content = mmap (0, len, PROT_READ | PROT_WRITE, MAP_SHARED, fd, 0)) == MAP_FAILED) /* error */
Other variations: N/A
